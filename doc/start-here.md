# chopper start here

If you're not sure where to begin, use this page.

---

## I want to use chopper right now

1. Read root `README.md` (quickstart + install/run).
2. Use `doc/quick-reference.md` for command lookup.
3. Copy from `doc/examples.md` or `doc/templates/`.

---

## I’m migrating existing aliases

1. Read `doc/migration.md`.
2. Use `doc/config-reference.md` while converting fields.
3. Use `doc/troubleshooting.md` for validation/debugging issues.

---

## I’m debugging behavior

1. Start with `doc/troubleshooting.md`.
2. Use `doc/cli-reference.md` to confirm invocation/env controls.
3. Use `doc/operational-spec.md` for exact semantics.

---

## I’m developing the codebase

1. Read `doc/architecture.md` for module/runtime flow.
2. Use `doc/testing.md` for local verification commands.
3. Use `doc/release-checklist.md` before shipping.
